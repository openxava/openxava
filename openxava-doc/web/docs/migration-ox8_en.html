How to update your project to the latest OpenXava?

Edit the pom.xml file in your project and change the value of the openxava.version property to point to the latest OpenXava version, thus:

<properties>
    <openxava.version>8.0</openxava.version>
    ...
</properties>

Rebuild your project with: mvn clean package

If your current version is prior to 8.0 you should follow the instructions in the section Migration from OpenXava 7.7.3 to OpenXava 8.0 first.

Finally, revise the instructions below to adapt your code.

Migration from OpenXava 7.7.3 to OpenXava 8.0

Java 17 as minimum version

Now Java 17 is the minimum supported version. If your project still uses Java 8 or 11 you should update it to compile with Java 17, 21 or 25. To do this modify the pom.xml:

<properties>
    <maven.compiler.source>17</maven.compiler.source>
    <maven.compiler.target>17</maven.compiler.target>
    ...
</properties>

Make sure you have a JDK 17 (or higher) installed on your system and configured in your IDE.

You can consult the official Java documentation for migration to Java 17 and if you use Java 8 the migration documentation from Java 8.
